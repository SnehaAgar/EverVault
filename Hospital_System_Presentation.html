Hospital Management System

EverVault - Smart Healthcare Scheduling

Project Overview
  • A full-stack web application for hospital equipment booking and queue management
  • Frontend: React.js with modern UI/UX design
  • Backend: Spring Boot REST API with MySQL database
  • Real-time queue management with priority-based scheduling
  • Equipment status tracking (Available, In-Use, Maintenance)
  • Comprehensive test coverage: JUnit (backend) + Jest (frontend)
Key Features
  • Patient Portal: Book appointments, view equipment availability
  • Admin Dashboard: Manage queues, approve bookings, call next patient
  • Priority System: Emergency > Urgent > Normal
  • Real-time Updates: Equipment status and queue positions
  • Validation: Past date prevention, maintenance equipment blocking
  • Procedure Duration Display: Shows estimated time for each equipment
Technology Stack
  • Frontend: React 19, Vite, CSS3
  • Backend: Spring Boot 3.2, Java 17
  • Database: MySQL 8.0
  • Testing: JUnit 5 (Backend), Jest + React Testing Library (Frontend)
  • API Documentation: Swagger/OpenAPI
  • Build Tools: Maven (Backend), npm (Frontend)

Key Components

  • Controller Layer: HospitalController - REST endpoints, exception handling
  • Service Layer: QueueService - Business logic, priority sorting, validation
  • Repository Layer: JPA Repositories - Database operations
  • Model Layer: Booking, Equipment, Priority, EquipmentStatus - Entity classes
  • Security: Input validation, past date check, maintenance status check

Demo Flow
  • 1. Start Backend: java -jar target/backend-0.0.1-SNAPSHOT.jar
  • 2. Start Frontend: npm run dev
  • 3. Open Swagger UI: http://localhost:8080/swagger-ui.html
  • 4. Show API Testing: Create booking, validate errors
  • 5. Show Frontend: Patient portal, Admin dashboard
  • 6. Show Queue Management: Call next patient, auto-status update
  • 7. Run Backend Tests: mvn test
  • 8. Run Frontend Tests: npm test -- --watchAll=false
